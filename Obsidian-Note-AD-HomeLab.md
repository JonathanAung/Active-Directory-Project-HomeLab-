## What Is This Project?

A home lab simulating a real corporate Windows domain environment. I set up:
- A Windows Server 2022 as a **Domain Controller** (the "boss" of the network — controls who can log in and access what)
- A Windows 10 machine as a **client** (like a regular employee's computer joined to the company network)
- **Splunk** on Ubuntu as a **SIEM** (Security Information & Event Manager — think of it as a security camera system that records everything happening on the network)
- **Kali Linux** as the **attacker machine** (used to simulate real hacker attacks in a safe environment)
- **Atomic Red Team** to run pre-built attack scripts that mimic real-world threats

## What I Learned

- How to build and configure a **Windows Active Directory domain** from scratch
- How to install and configure **Splunk** to ingest Windows Event Logs (the "security camera footage")
- How to use **Kali Linux** and **Atomic Red Team** to simulate attacks like brute force and lateral movement
- How to read **Windows Event IDs** — these are codes Windows generates for every action:
  - `4625` = Failed login attempt (someone typed the wrong password)
  - `4624` = Successful login
  - `4720` = A new user account was created
  - `4732` = A user was added to a privileged group
- How to spot **Indicators of Compromise (IOCs)** in Splunk — patterns that suggest an attack is happening
- How **NAT networking** works in VirtualBox and how to assign static IPs
- Linux CLI basics: `ip a`, `sudo nano`, `sudo netplan apply`

## Key Concepts (Plain English)

**Active Directory** = The company's "employee directory + security guard." It manages who exists on the network, what they can access, and enforces password policies.

**SIEM (Splunk)** = A system that collects logs from all machines and lets you search/alert on suspicious activity. SOC analysts use this daily.

**Brute Force Attack** = Trying thousands of passwords automatically until one works. Atomic Red Team simulates this so I can see what it looks like in Splunk.

**Lateral Movement** = After breaking into one machine, an attacker tries to "move sideways" to other machines on the network. This is what makes breaches dangerous.

**Telemetry** = Data/logs generated by activity on a system. More telemetry = better visibility for defenders.
